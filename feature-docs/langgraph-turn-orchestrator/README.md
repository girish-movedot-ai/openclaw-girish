# LangGraph Turn Orchestrator

This folder contains all internal and experimental documents for the LangGraph turn-orchestrator feature.

## Structure

- `spec/` contains the source documents provided by the human operator
- `deliverables/` contains the repo-discovery and implementation-planning documents generated by the first agent

Current source spec set:

- `spec/01-design-intent.md`
- `spec/03-operating-mind-spec.md`

## Intended workflow

1. Read everything in `spec/`
2. Generate the requested documents into `deliverables/`
3. Review `deliverables/00a-success-criteria.md` first and treat it as the highest-weight document in the entire feature package
4. Review and commit the deliverables
5. Use a separate agent to implement the feature from the reviewed deliverables

## Priority

The implementation priority order for this feature is:

1. `deliverables/00a-success-criteria.md`
2. `deliverables/03-failure-register.md`
3. `deliverables/04-implementation-spec.md` and `deliverables/00b-system-profile.md`
4. the remaining deliverables

This feature is not considered complete if the system only proves LangGraph wiring. The target is a real reconstruction-driven orchestrator that produces stable, fast, tool-aware behavior across turns.
