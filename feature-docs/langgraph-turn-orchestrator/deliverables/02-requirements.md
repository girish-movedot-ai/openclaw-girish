# Requirements: LangGraph Turn-Orchestrator Replacement

## Priority

These requirements are subordinate to `00a-success-criteria.md`.

If an implementation satisfies a requirement here but fails the success criteria, the feature is not complete.

| ID      | Requirement                                                                                                                                                                                            | Source                 | Testable Criterion                                                                                                                                            | Priority |
| ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- |
| FR-001  | The caller-facing turn entry point must remain `runEmbeddedPiAgent(params): Promise<EmbeddedPiRunResult>`.                                                                                             | US-001                 | Existing direct callers compile and run without changing their call sites or expected return shape.                                                           | Must     |
| FR-002  | Orchestration mode must resolve through an explicit routing path that adds LangGraph without breaking the existing embedded runtime path.                                                              | US-001                 | A turn can be routed to legacy embedded or LangGraph by explicit configuration, and the embedded path still works unchanged.                                  | Must     |
| FR-003  | The LangGraph `respond` path must finish the turn without host execution and must return normal `EmbeddedPiRunResult.payloads` plus `meta`.                                                            | US-002                 | A respond-only turn emits no execution request and returns at least one non-error text/media payload or an intentionally empty payload set with valid `meta`. | Must     |
| FR-004  | The LangGraph `ask_clarification` path must return a blocking clarification reply and must not emit a host execution request.                                                                          | US-004, NS-004         | A clarification turn returns a user-visible question, marks the turn as blocked in LangGraph output, and emits no execution request.                          | Must     |
| FR-005  | The LangGraph `execute` path must delegate all real execution through existing OpenClaw host surfaces instead of creating a second execution stack.                                                    | US-003                 | Execution requests are satisfied only by the TS host adapter using the existing tool/exec/approval surfaces already used by embedded turns.                   | Must     |
| FR-006  | Approval continuation in LangGraph mode must resume the same turn through `resume_turn`; it must not reuse the current async follow-up-only approval pattern as the LangGraph completion mechanism.    | US-003                 | After approval, the same LangGraph turn resumes and produces the final response through the original turn flow.                                               | Must     |
| FR-007  | The graph verify step may request at most one retry back to diagnosis/execution planning.                                                                                                              | US-003                 | A failing execution can trigger one re-diagnosis/retry loop; a second retry attempt is rejected and the turn terminates as failure.                           | Must     |
| FR-008  | Every turn in LangGraph mode must reconstruct a stable operating mind from persisted state instead of acting as a generic fresh assistant.                                                             | SC-005, SC-006         | The assistant preserves identity, user model, permissions, and task posture across turns without repeated reminders.                                          | Must     |
| FR-009  | The reconstruction phase must deterministically rebuild who the assistant is, who the user is, what the assistant can do, what permissions it has, and what active task context matters for this turn. | SC-005, SC-006, SC-007 | Logs, traces, or state artifacts show a structured reconstruction step with those inputs present and used.                                                    | Must     |
| FR-010  | The decision policy must consistently choose between act, ask, escalate, and execute based on reconstructed state rather than ad hoc heuristics or placeholder routing.                                | SC-008                 | A clear task leads to action, an ambiguous task leads to clarification, and a risky task respects approval policy.                                            | Must     |
| FR-011  | Final replies in LangGraph mode must be user-facing and must not expose raw sender metadata, internal transport objects, or placeholder debug dumps during normal operation.                           | SC-005, SC-010         | Normal replies read like assistant responses rather than internal object renderings.                                                                          | Must     |
| FR-012  | Medium-complexity tasks must remain fast enough to feel usable after adding reconstruction and policy phases.                                                                                          | SC-009                 | Test runs show structured LangGraph behavior without obvious over-deliberation or severe latency regression on medium tasks.                                  | Must     |
| FR-013  | Turn completion must write back updated structured state so later turns benefit from newly learned preferences, task progress, and operating context.                                                  | SC-006                 | After one turn updates preferences or task state, later turns use that information without needing restatement.                                               | Must     |
| FR-014  | The LangGraph implementation must not be considered complete if it only proves routing, sidecar startup, and a technically valid reply.                                                                | SC-005                 | Placeholder, echo, or stub behavior fails acceptance even if the sidecar path runs.                                                                           | Must     |
| IR-001  | Every field in `RunEmbeddedPiAgentParams` must have an explicit adapter disposition: mapped into graph request, retained on host only, or recorded as a Gap.                                           | US-001                 | The implementation includes a field-by-field mapping table with no omitted input fields.                                                                      | Must     |
| IR-002  | Every field in `EmbeddedPiRunResult` and nested `EmbeddedPiRunMeta` / `EmbeddedPiAgentMeta` must have an explicit source from graph output, host execution result, or adapter synthesis rule.          | US-001                 | The implementation includes a field-by-field output mapping table with no omitted output fields.                                                              | Must     |
| IR-003  | Non-serializable current inputs (`AbortSignal`, callbacks, enqueue function, mutable refs) must stay host-side and must not be sent over the TS↔Python RPC boundary.                                   | US-001                 | The RPC request/response payloads are JSON-serializable and the host adapter preserves callback-driven behavior locally.                                      | Must     |
| FHR-001 | Unexpected orchestration mode values must not silently coerce to an arbitrary path.                                                                                                                    | NS-001                 | Invalid values are either rejected or mapped by one explicit fallback rule that is covered by tests.                                                          | Must     |
| FHR-002 | If the sidecar is unavailable before a new turn starts, routing policy must either fall back to legacy embedded or return an explicit startup failure; it must not hang.                               | NS-002                 | A failed pre-turn health/invoke check finishes promptly and does not leave the caller waiting indefinitely.                                                   | Must     |
| FHR-003 | Once a turn has entered LangGraph mode, sidecar failure must surface as a LangGraph-turn failure and must not fall back to legacy embedded mid-turn.                                                   | NS-003, NS-005         | A crash after `invoke_turn` started returns a failure/error result and no legacy execution occurs for that turn.                                              | Must     |
| FHR-004 | RPC timeouts for `invoke_turn` and `resume_turn` must fail the turn loudly and must not block the gateway indefinitely.                                                                                | NS-005                 | Timeout paths emit structured failure output and the caller regains control without waiting beyond the configured timeout.                                    | Must     |
| FHR-005 | Approval state and graph resume state must have an explicit persistence contract; supported restart cases must resume correctly, unsupported restart cases must fail explicitly.                       | US-003, NS-005         | Restart behavior is tested in both “checkpoint exists” and “checkpoint missing” cases, with no silent drop of approval state.                                 | Must     |
| NFR-001 | LangGraph routing must preserve one-turn-at-a-time session behavior that the current runner enforces with session lanes.                                                                               | US-001                 | Two concurrent turns for the same session are serialized or explicitly rejected by one documented rule.                                                       | Must     |
| NFR-002 | LangGraph execution must emit observability fields that can be correlated with existing run lifecycle logs using the current `runId` as the trace key source.                                          | NS-005                 | Logs/events include at least run/trace id, agent id, session id, orchestration mode, graph node, intent, and terminal state.                                  | Must     |
| NFR-003 | The default routing mode for this feature must remain the current legacy embedded path until LangGraph is explicitly selected.                                                                         | US-001, NS-002         | A system with no LangGraph-specific config continues to run the current embedded path.                                                                        | Must     |
