# Failure Register: LangGraph Turn-Orchestrator Replacement

## Taxonomy Applicability

| Category                     | Applicable? | Reason                                                                                                                                            |
| ---------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| C1 Identity/Access           | No          | The feature replaces turn orchestration, not auth or caller identity. Existing identity/session fields still pass through the same turn boundary. |
| C2 Config/Environment        | Yes         | Routing mode, sidecar startup, Python availability, and restart behavior are all environment-sensitive.                                           |
| C3 State Machine/Concurrency | Yes         | The new graph adds explicit node routing, bounded retry, approval resume state, and same-session concurrency rules.                               |
| C4 Contract/Boundary         | Yes         | The highest-risk work is the TS host â†” Python sidecar contract plus exhaustive input/output mapping.                                              |
| C5 Data/Artifact Pipeline    | No          | No new long-lived data pipeline is being introduced for v1.                                                                                       |
| C6 Security/Privacy          | No          | The experimental scope does not add a new external network trust boundary; the sidecar is local-only.                                             |
| C7 Reliability/Observability | Yes         | Sidecar lifecycle, timeout handling, crash detection, and failure visibility are primary bug risks.                                               |
